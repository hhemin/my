# 📱 Temi Go Pro —— 智能室内配送机器人控制平台

项目简介
Temi Go Pro 是一款专注于室内封闭场景（如酒店、写字楼、医院）的智能配送机器人控制应用。用户通过 App 一键下单后，系统会调度实体机器人前往指定地点取货（揽件），并实时追踪其运动轨迹，最终送达目标收件地点。App 充当了"下单器 + 实时监控仪表盘"的双重角色，实现了配送任务的闭环管理。

技术栈

  • 跨端框架：Vue 3 + uni-app + TypeScript
  • 样式引擎：UnoCSS
  • 实时通信：MQTT（设备状态同步与指令下发）
  • 可视化引擎：Three.js（结合地图 SDK 构建 3D/2D 实时轨迹）
  • 混合架构：原生壳 + WebView（H5 通用模块桥接）
  • 设计提效：MCP（Model Context Protocol）读取 Figma + Skills 约束 AI 代码生成
  • 目标平台：iOS、Android、鸿蒙、微信小程序（已过审）

个人职责与核心难点攻克

  • 项目主导与架构设计：在此项目中担任技术主导角色，负责整体项目主体搭建、技术选型及核心难点攻关，并主导了与硬件端（机器人）的 MQTT 通信协议设计。
  • 攻克难点：抽取通用 Three.js 地图 H5 模块：
    针对配送场景中复杂的机器人实时定位、路径规划与 3D 轨迹展示需求，若使用 uni-app 原生 Canvas 或地图组件，在多端（尤其鸿蒙与小程序）下性能与兼容性风险极高。为此，我独立抽离并封装了一套基于 Three.js 的通用地图可视化 H5 应用。该模块通过 WebView 嵌入 App，通过 JSBridge 与原生层通信，完美承载了设备位置渲染与动态路径绘制。
    更关键的是，该 H5 模块被设计为"一次开发，随处复用"的通用组件，后续团队其他机器人相关项目仅需通过 WebView 导入同一份 H5 链接即可直接复用地图与轨迹能力，极大降低了同类项目的开发周期与维护成本。
  • 多端兼容与混合通信：沿用成熟的"WebView 承载复杂业务"策略，将配送逻辑、MQTT 长连接状态机（待取件→运送中→已送达）置于 H5 容器中处理，利用 uni-app 条件编译处理原生 UI层，彻底屏蔽了 iOS、Android、鸿蒙及小程序在底层 API 上的差异。特别是针对微信小程序环境，通过 WebView 与小程序原生层的特殊桥接，确保了配送地图在小程序端的流畅运行（目前已通过微信审核）。

项目成果

  • 已完成 iOS、Android、鸿蒙 App 端的主体开发，微信小程序版本已成功通过官方审核，具备上线条件。
  • 成功沉淀出一套 "Three.js 通用地图 H5 脚手架"，成为团队内部可复用的标准化资产。
  • 通过 WebView 混合架构与通用模块抽取，在保证多端 UI 高度一致的同时，将复杂业务的兼容性问题隔离在 H5 层，为项目后续快速迭代奠定了坚实基础。
