# 📱 Temi Care —— 智能机器人告警与远程关爱App（第一个 Ai 工程化项目）

项目简介
Temi Care 是一款配合智能服务机器人使用的移动端应用，主要面向养老、安防等场景。当机器人检测到跌倒、异常告警等突发事件时，App 会实时推送通知，用户可立即查看事件详情，并通过视频通话远程确认现场状况，实现"告警 - 查看 - 沟通"闭环。

考虑当时任务时间比较紧迫，自己任务量较大，该项目由我独立完成。加上对Ai有一定了解，就大胆放手去干，最终如期交付。

技术栈

  • 框架：Vue 3 + uni-app + TypeScript
  • 样式：UnoCSS
  • 通信：MQTT、声网（Agora）RTC
  • 混合开发：原生 + WebView（H5 桥接）
  • 工具链：MCP（Model Context Protocol）读取 Figma 设计稿与接口文档，Skills 约束 AI 生成代码

个人职责与核心难点攻克

  • 独立完成全部开发工作，从静态页面到整体交付历时约 2 个月，后续迭代排期 1~2 周。
  • 多端兼容策略：项目需同时支持 iOS、Android 及鸿蒙（HarmonyOS）。面对鸿蒙对 nvue、部分权限的兼容性问题，采用"原生壳 + WebView 承载复杂业务"的混合架构，将视频通话和地图模块抽离为 H5 页面，通过 JSBridge 实现 App 与 Web 的双向通信，有效规避 uni-app 原生实现中的端差异，大幅提升开发效率与稳定性。
  • 视频通话模块：基于声网 SDK 封装，通过 WebView 集成，解决 uni-app 原生插件在多端表现不一致的问题，保证通话质量与兼容性。
  • 地图模块：将地图相关逻辑抽取为独立 H5 应用，供 App 及外部 Web 复用，减少重复开发。
  • 无感登录：利用设备 CID 实现自动身份识别，提升用户体验。
  • AI 辅助提效：引入 MCP 协议自动解析 Figma 设计稿生成页面结构，并读取接口文档快速完成数据绑定；同时定制 Skills 规范统一页面风格，减少 AI 幻觉，使基础 CRUD 逻辑可完全交由 AI 生成，显著缩短静态页面开发时间（约 1 周完成 15+ 页面）。

项目成果

  • App 已成功上架苹果 App Store、主流 Android 应用市场及鸿蒙应用市场，搜索"Temi Care"即可下载。
  • 覆盖事件通知、事件详情回放、实时视频通话、地图定位等完整功能，页面数量 15+。
  • 通过混合开发架构，在有限周期内高质量交付，且为后续多端维护提供了可复用的技术方案。
