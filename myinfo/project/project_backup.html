# 📱Temi Care —— 智能机器人告警与远程关爱App （第一个Ai工程化项目）

项目简介
Temi Care 是一款配合智能服务机器人使用的移动端应用，主要面向养老、安防等场景。当机器人检测到跌倒、异常告警等突发事件时，App 会实时推送通知，用户可立即查看事件详情，并通过视频通话远程确认现场状况，实现"告警 - 查看 - 沟通"闭环。

考虑当时任务时间比较紧迫，自己任务量较大，该项目由我独立完成。加上对Ai有一定了解，就大胆放手去干，最终如期交付。

技术栈

  • 框架：Vue 3 + uni-app + TypeScript
  • 样式：UnoCSS
  • 通信：MQTT、声网（Agora）RTC
  • 混合开发：原生 + WebView（H5 桥接）
  • 工具链：MCP（Model Context Protocol）读取 Figma 设计稿与接口文档，Skills 约束 AI 生成代码

个人职责与核心难点攻克

  • 独立完成全部开发工作，从静态页面到整体交付历时约 2 个月，后续迭代排期 1~2 周。
  • 多端兼容策略：项目需同时支持 iOS、Android 及鸿蒙（HarmonyOS）。面对鸿蒙对 nvue、部分权限的兼容性问题，采用"原生壳 + WebView 承载复杂业务"的混合架构，将视频通话和地图模块抽离为 H5 页面，通过 JSBridge 实现 App 与 Web 的双向通信，有效规避 uni-app 原生实现中的端差异，大幅提升开发效率与稳定性。
  • 视频通话模块：基于声网 SDK 封装，通过 WebView 集成，解决 uni-app 原生插件在多端表现不一致的问题，保证通话质量与兼容性。
  • 地图模块：将地图相关逻辑抽取为独立 H5 应用，供 App 及外部 Web 复用，减少重复开发。
  • 无感登录：利用设备 CID 实现自动身份识别，提升用户体验。
  • AI 辅助提效：引入 MCP 协议自动解析 Figma 设计稿生成页面结构，并读取接口文档快速完成数据绑定；同时定制 Skills 规范统一页面风格，减少 AI 幻觉，使基础 CRUD 逻辑可完全交由 AI 生成，显著缩短静态页面开发时间（约 1 周完成 15+ 页面）。

项目成果

  • App 已成功上架苹果 App Store、主流 Android 应用市场及鸿蒙应用市场，搜索"Temi Care"即可下载。
  • 覆盖事件通知、事件详情回放、实时视频通话、地图定位等完整功能，页面数量 15+。
  • 通过混合开发架构，在有限周期内高质量交付，且为后续多端维护提供了可复用的技术方案。

# 📱 Temi Go Pro —— 智能室内配送机器人控制平台

项目简介
Temi Go Pro 是一款专注于室内封闭场景（如酒店、写字楼、医院）的智能配送机器人控制应用。用户通过 App 一键下单后，系统会调度实体机器人前往指定地点取货（揽件），并实时追踪其运动轨迹，最终送达目标收件地点。App 充当了"下单器 + 实时监控仪表盘"的双重角色，实现了配送任务的闭环管理。

技术栈

  • 跨端框架：Vue 3 + uni-app + TypeScript
  • 样式引擎：UnoCSS
  • 实时通信：MQTT（设备状态同步与指令下发）
  • 可视化引擎：Three.js（结合地图 SDK 构建 3D/2D 实时轨迹）
  • 混合架构：原生壳 + WebView（H5 通用模块桥接）
  • 设计提效：MCP（Model Context Protocol）读取 Figma + Skills 约束 AI 代码生成
  • 目标平台：iOS、Android、鸿蒙、微信小程序（已过审）

个人职责与核心难点攻克

  • 项目主导与架构设计：在此项目中担任技术主导角色，负责整体项目主体搭建、技术选型及核心难点攻关，并主导了与硬件端（机器人）的 MQTT 通信协议设计。
  • 攻克难点：抽取通用 Three.js 地图 H5 模块：
    针对配送场景中复杂的机器人实时定位、路径规划与 3D 轨迹展示需求，若使用 uni-app 原生 Canvas 或地图组件，在多端（尤其鸿蒙与小程序）下性能与兼容性风险极高。为此，我独立抽离并封装了一套基于 Three.js 的通用地图可视化 H5 应用。该模块通过 WebView 嵌入 App，通过 JSBridge 与原生层通信，完美承载了设备位置渲染与动态路径绘制。
    更关键的是，该 H5 模块被设计为"一次开发，随处复用"的通用组件，后续团队其他机器人相关项目仅需通过 WebView 导入同一份 H5 链接即可直接复用地图与轨迹能力，极大降低了同类项目的开发周期与维护成本。
  • 多端兼容与混合通信：沿用成熟的"WebView 承载复杂业务"策略，将配送逻辑、MQTT 长连接状态机（待取件→运送中→已送达）置于 H5 容器中处理，利用 uni-app 条件编译处理原生 UI 层，彻底屏蔽了 iOS、Android、鸿蒙及小程序在底层 API 上的差异。特别是针对微信小程序环境，通过 WebView 与小程序原生层的特殊桥接，确保了配送地图在小程序端的流畅运行（目前已通过微信审核）。

项目成果

  • 已完成 iOS、Android、鸿蒙 App 端的主体开发，微信小程序版本已成功通过官方审核，具备上线条件。
  • 成功沉淀出一套 "Three.js 通用地图 H5 脚手架"，成为团队内部可复用的标准化资产。
  • 通过 WebView 混合架构与通用模块抽取，在保证多端 UI 高度一致的同时，将复杂业务的兼容性问题隔离在 H5 层，为项目后续快速迭代奠定了坚实基础。

# 小虎智学App —— AI数字人智慧课堂

关键词速览： Vue3 uni-app TypeScript 双WebSocket UnoCSS+纯原生CSS 数字人 科大讯飞 平板适配 从0到1架构

项目简介
一款面向K12教育的AI智能学习应用，通过数字人虚拟老师为学生提供沉浸式课堂体验。核心功能包括：作文拍照智能批改与评分报告、数字老师动态切换黑板内容（文字高亮讲解/图片/视频）、配合语音合成进行生动讲解，打造“真人教师”般的在线课堂。已上架iOS/安卓应用市场，完美适配平板模式。

核心技术亮点

技术领域 具体实践
双WebSocket架构 控制通道（黑板指令/高亮/切换）+ 语音通道（音频流），实现前后端解耦
黑板多模块渲染引擎 统一解析器处理三种模块：文字高亮讲解、纯图片展示、视频播放，支持无缝切换与连贯动画
纯原生CSS + UnoCSS 放弃UI框架，手写全部样式，精准控制儿童化UI（明亮配色、大圆角、柔和动效），平板大屏响应式适配
语音交互 集成科大讯飞API，实现语音输入与指令识别，降低低龄用户操作门槛
作文批改闭环 拍照上传 → OCR识别 → AI评分 → 数字老师逐句讲解 → 生成报告，完整闭环
全局奖励系统 封装星星弹出动画组件，学习过程中适时正向激励，提升学习粘性
TypeScript全栈 严格类型定义覆盖API、WebSocket消息、黑板指令，保障重构后代码健壮性

个人职责与难点攻克

  • 项目重构与主导：该项目原由其他同事开发，后因团队变动由我全盘接手并主导架构升级，完成从“前端控制提示词”到“后端主导AI编排”的架构演进，将混乱的单通道通信重构为清晰的双通道设计。

  • 双WebSocket通道架构设计（核心难点）
    早期单通道中，AI返回的讲解文本、黑板指令、音频流混合下发，前端需自行拆解，逻辑混乱且调试困难。我主导设计双WebSocket长连接架构，将控制流与音频流分离，前端职责回归“纯展示与交互”，大幅降低维护复杂度。

  • 黑板多模块渲染引擎（核心难点）
    根据AI指令实时切换三种展示形态：文字高亮讲解、纯图片展示、视频播放。设计一套黑板指令解析器统一处理各模块渲染逻辑，保障切换动画连贯性，使教学内容呈现更加生动。

  • 儿童化UI与平板适配
    鉴于主流UI框架风格过于“通用/商务”，采用UnoCSS + 原生CSS手写全部样式，精准控制圆角、配色与动效。针对iPad及安卓平板进行专门响应式适配，确保大屏布局完美呈现（解决uni-app rpx适配差异等兼容问题）。

  • 作文批改闭环
    从拍照上传 → OCR识别 → AI评分 → 数字老师逐句讲解，完成了完整的“提交-批改-报告”闭环，涵盖图像上传、结果解析、报告生成全流程。

项目成果

  • ✅ App已成功上架苹果App Store及各大安卓应用市场，平板模式体验优秀
  • ✅ 双WebSocket架构升级，课堂互动更流畅
  • ✅ 沉淀可复用资产：黑板多模块渲染引擎 + 儿童化CSS组件库

# 🎨 Ai教育 —— AI绘画互动故事课堂

关键词速览： Vue3 TypeScript Pinia 纯原生CSS Stable Diffusion GPT 提示词工程 响应式适配 从0到1架构

项目简介
一款面向儿童的AI互动教育应用，以“故事+绘画”为主线，包含两大核心模块：绘画工坊（分步引导儿童完成SD文生图/图生图）和故事乐园（儿童版GPT，通过提示词约束实现儿童化对话）。项目从0到1独立完成全栈前端架构与AI服务整合。

核心技术亮点

技术领域 具体实践
AI服务编排 整合GPT + Stable Diffusion双AI接口，设计统一调度层完成“对话→Prompt组装→图片生成”全链路
SD绘画工作流 分步选择场景/角色/动作 → 自动组装结构化Prompt（含正/负向提示词）→ 调用SD API生成；支持图生图 + 涂鸦预处理
儿童版GPT 通过System Prompt严格控制语言风格，设计话题引导机制，使AI以“温柔耐心”的儿童化口吻稳定输出
纯原生CSS 放弃UI框架，手写全部样式，精准控制儿童化UI（明亮配色、大圆角、柔和动效），响应式适配手机/平板/PC
TypeScript全栈 严格类型定义覆盖API请求、AI响应解析、Prompt模板，保障代码健壮性
工程化设计 模块化封装SD Prompt组装器、AI调度服务、图片预处理器，具备高可扩展性（可替换AI服务商）

个人职责与难点攻克

  • 从0到1架构搭建：独立完成项目主体架构设计，涵盖技术选型、目录结构、状态管理（Pinia）及接口层封装。
  • SD分步引导生成逻辑：针对儿童无法编写复杂Prompt的痛点，设计“场景→角色→动作”分步选择器，前端自动组装为SD所需Prompt参数，辅以图生图涂鸦预处理（尺寸裁剪+对比度增强），提升生成成功率。
  • 故事乐园提示词工程：多轮调优System Prompt，约束GPT输出风格为儿童化、安全化，并植入主动引导机制，保障对话可延续性。
  • 儿童化UI高还原交付：基于Figma设计稿，采用纯原生CSS手写全部样式，无UI框架依赖，确保视觉风格童趣、准确、适配多端。

项目成果

  • ✅ 完成“绘画工坊”与“故事乐园”完整闭环，儿童可在听故事后参与绘画创作
  • ✅ 沉淀可复用资产：儿童化CSS组件库 + SD Prompt组装模板
